Symptoms

  1. Added new-clean MonoForAndroid project to existing VS solution
    1. Run/Debug button is disabled.

  2. Visited older test project
    1. Run/Debug button is enabled.
      Deployment is possible and in Build Log in VS there is Deploying to device 100%
      Nothing is deployed on device, app never starts.
      In adb.log there were some errors (GPS related and some other,

Suspicion

Before Christmas holidays there was a lot done with Mono Mobile API. Tests and development. In order to extend Mono Mobile API ,

  • reverse-engineered API with ILSpy (just to see source)
    ILSpy - new tool to put on the list... This SwissKnife extracts even comments?!?!?!
  • spilt it into projects with help of Project Linker
  • Added Windows Phone rudimentary API for Geolocation.
    But the test could not be made without Mango, cos emulator that comes with Mango SDK has GeoLocation "mocking"/simulation.
    ... and (shame) at that point team had no devices to test with...
    Thanks to my Bro www.studio-sac.net the team has right now several iPhones and Samsung Galaxy II S...

Assumptions are that Mango installation messed something up with VS and M4A.

Resolution

  1. Reinstalled latest version of Mono For Android
  2. Run CCleaner registry clean
  3. rebooted - did not try without it, maybe it could work
