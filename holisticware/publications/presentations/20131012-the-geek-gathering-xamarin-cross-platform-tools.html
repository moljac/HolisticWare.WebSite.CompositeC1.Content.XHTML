Xamarin.iOS 02

MVC pattern

  • Model View Controller
    most important design pattern is MVC
    Model View Controller
    • Model
      Encapsulates the data of an app and defines a logic
    • View
      Is an object that users can see, knows how to draw itself and might respond to user actions
    • Controller
      Intermediary between one or more View objects and one or more Model objects.

Xamarin.iOS 04

Delegation

  • Acting on behalf on another object
  • Used when an object receives many events and it wants the same Object to handle each of those events.
  • C# == Events

Xamarin.iOS 05

Protocol

  • Enabling communication between objects not related by inheritance
  • C# == Interfaces (abstract classes)

Xamarin.iOS 06

Notification Center

  • Notifying Interested Observers of an Event
  • Every object can register an event
    Every object can subscribe to an event

Xamarin.iOS 07

Target Action

  • Encapsulating a message to be sent when event occurs
  • Used when you have close relationship between two objects
  • ViewController and on of its Views (UIButton)

Xamarin.iOS 08

Key – Value Observing

  • Notifying an observer when a value changes
  • Allows an object to observe a property of another object

Xamarin.Android

Components in detail 01

  • Activities
    application component that provides a interaction screen (or not)
    Orchestrates a UI view
    Applications are composed of 1-to-Many activities
    One activity marked as main and shown first upon launch
  • Intents
    messaging facility for late run-time binding between components in the same or different applications
    Eventing mechanism
    Intent objects are passive data that is of interest to the component that is receiving the intent
    Filterable

Xamarin.Android

Components in detail 02

  • Services
    application component that can perform long-running operations in the background and does not provide a user interface
    Long lasting background work
    No UI
    Like Windows services
    Possible to bind to an ongoing service and communicate via exposed interface
    Runs in main application process but doesn’t block other components or UI
