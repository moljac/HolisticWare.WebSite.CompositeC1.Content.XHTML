async c# v.5 - .net 4.5

public async Task<int> ExampleMethodAsync()
{
	var httpClient = new HttpClient();
	
	// The following line activates GetStringAsync, which is an asynchronous method.
	Task<string> contentsTask = httpClient.GetStringAsync("http://msdn.microsoft.com");
	
	// While the task is active, you can do other work.
	ResultsTextBox.Text += "Doing other work before awaiting contentsTask. . . . .\n";
					
	// When you await contentsTask, execution in ExampleMethodAsync is suspended,
	// and control returns to the caller, StartButton_Click.
	string contents = await contentsTask;
			
	// After contentTask completes, you can calculate the length of the string.
	int exampleInt = contents.Length;
					
	//// You can combine the previous sequence into a single statement.
	//int exampleInt = (await httpClient.GetStringAsync("http://msdn.microsoft.com")).Length;
							
	ResultsTextBox.Text += "Preparing to finish ExampleMethodAsync.\n";
	// After the following return statement, any method that's awaiting
	// ExampleMethodAsync (in this case, StartButton_Click) can get the integer result.
	
	return exampleInt;
}

Testing mobile apps 1

  • On mobile quality is hard, thus testing is very important.
  • Mobile users are picky/spoiled
  • the store: bugs hurt application lowering the ratings, forcing users to move to other applications
  • enterprise apps: engagement goes down and project fails
  • Sheer numbers - permutations - complexity
    • Android:
      5+ OS versions
      devices: n * 10^2 (8 vendors)
      27 screen sizes
      20 languages
    • iOS:
      3 OS versions (iOS 6, 5, 4)
      8 devices, but expensive
      20 languages

Testing mobile apps 2

Development challenges:

  • fast moving tech
  • Regressions (dev cycle 6 weeks on desktop 6 months)
  • store influence (feedback, rating)

Testing options:

  • unit testing (no full coverage - only libraries/utilities)

Internals

XTC

  • based on Calabash http://calaba.sh/
    open-source automated UI testing script framework

  • and Martin Fowler’s Business Readable Domain Specification Language
  • Similar ideas can be found in NBehave,
    .net implementation for BDD - Behavior Driven Development and Testing.

Xamarin Test Cloud 4

Calabash

  • min purpose is to support BDD (Behavior Driven Development)
  • that tests are written in plain text only
  • BDD recommends use of: specifications, use-cases, feature-lists
  • BDD discourages: that anyone (management) writes tests
  • test should be readable and understandable by project and team members
  • should improve and encourage communication
  • open-source scripting framework written in objective-c, java, ruby
