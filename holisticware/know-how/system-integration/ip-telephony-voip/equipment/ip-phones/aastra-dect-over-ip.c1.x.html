Aastra DECT over IP (SIP DECT)

AASTRA RFP (RPF - Radio Fixed Part, something like base station in DECT terminology)

  1. connect AASTRA RFP to network, from back side of the device read and write down MAC address which we will use to find AASTRA on our network.
  2. From internet download latest version of java configuration utility and TFTP boot image which is mandatory
  3. Copy TFTP boot image that was downloaded from the internet to TFTP server
  4. In main window of OMC tool enter MAC of RFP then click to ~list configuration~ to get current configuration
  5. Wait few seconds for RFP respond. If MAC is OK, configuration is listed. For new configuration check step 6.
  6. OMM tool short description:
    1. use local configuration : no for DHCP, YES for entering network parameters manually
      (DHCP requires IP and TFTP reservation at DHCP server!)
    2. IP address & netmask - ip address of RFP with netmask
    3. TFTP address - address of TFTP server. On TFTP server put configuration file for RFP
      (Root is recommended, but not necessary)
    4. TFTP file name - name of configuration file that is on TFTP server (with address specified above).
    5. OMM IP address - IP address of Open Mobility Manager, in single RFP scenario must be same as IP of RFP (step 6a)
    6. Router address - leave blank
  7. when entering data is complete click to send configuration and wait few seconds
  8. It may take a while to RFP download and install TFTP image (watch out for LED progress).
  9. It may require manual power on/off
  10. If everything is OK and green LED is active, start browser and point it to the IP address we set up for OMM
  11. Inside OMM (web configuration utility)
    1. under the system set up all important parameters, PARK license which can be found on product CD is mandatory
    2. under Radio Fixed Parts create new cluster with our device (AASTRA RFP)
    3. under Portable Radio Parts (place where we set up DECT phones)
    4. enter all necessary DECT phone settings and turn on option that allows new phone to be registered
  12. On DECT phone, settings - New Subscription - enter PARK license
    (numerical format, can be found under settings, or Radio Parts right center of the screen)

appendix:

  • Red LED (booter) - ON - waiting for link up
  • Red LED (booter) - flashing 0.5Hz - Launching a DHCP request and waiting for an DHCP offer
  • RED LED (booter) - flashing 2,5Hz - Downloading the application image from TFTP
  • Orange LED (application) - ON - launching DHCP request and waiting for DHCP reply
  • Green LED (application) - green flashing 0,5Hz - RFP is initializing internal components
  • Green LED (application) - green flashing 1Hz - RFP is trying to connect to the OMM
  • Green LED (application) - green flashing (2 sec. ON, 0,5sec OFF) - The DECT part of RFP does not work, either not configured or not synchronized with other RFPs'
  • Green LED (application) - ON - RFP is up and running
  • On RFP L32 IP upper LED is unused, on L42 IP models there is green led for WLAN activity
