Partial Classes

Usage

Partial classes (partial keyword) are heavily used across .net framework for code separation across multiple source code files (.cs).
This enables:

  • concurrent work of multiple developers on the same class
  • finer control over the build process without using #defines / #ifdefs constructs which are considered as unclean coding in c/c++
  • separation of concepts

Note:
Partial classes are additive/cumulative - the code (attributes too) defined in one adds up to the definition of the whole class.
