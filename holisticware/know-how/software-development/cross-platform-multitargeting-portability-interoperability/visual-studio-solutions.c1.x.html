Visual Studio Solutions

Locations

Recommended setup is to have separate partition for development to put the root directory of projects and solutions as close as possible to the root of partition (Filesystem).
Due to the project structure and naming recommendations projects located in standard user folders can cause problems with long pathnames.
Sample developer setup: 

	d:\src\Core_2011
	d:\src\HolisticWare
	d:\src\HolisticWare\HolisticWare.SomeProject\

Where Core_2011 is project with utilities used across other project.

This means if using relative paths in Solution Files (.sln) developers do not need extra partition. Only restriction is to maintain consistent relative positions/paths of project worked on.

    Project order:
  1. Business Logic Layer (BL) libraries (dlls, assemblies)
    numbers 2xx
  2. Presentation Layer (PL) libraries (dlls, assemblies)
    Used for reusable UI components
    (2012-03 MonoTouch and Mono for Android do not support) numbers 5xx
  3. Presentation Layer (PL) entry points (main functions)
    Used for reusable UI components
    (2012-03 MonoTouch and Mono for Android do not support)
    numbers 8xx

Creating Projects - Defining Visual Studio Solution

In this sample there are

  1. only 2 architectural Layers:
    1. Business Logic (BL) and
    2. Presentation Layer (PL) (User Interface UI).
  2. several platform depending on application type:
    1. Mobile Clients (mClnt)
      Windows Phone (WP), Mono for Android (M4A), (MonoTouch optionally from MacOSX added)
    2. RIA Clients (rClnt)
      Silverlight (SL)
    3. Desktop Clients (dClnt)
      Windows Forms (WF) and/or Windows Presentation Foundation (WPF)
    4. Server (Srvr)
      ASP.net

In late 2011 (2011-09) smallest common subset (SCS) of functionalities was for Windows Phone, so WP platform is used as source project for Project Linker to other projects.

Try using recommendations: Project and Solution Structure on $Vendor$ projects.
Pseudo howto:

  1. For each layer in {BL, PL}
      For each platform in {WP, M4A, SL, WF, WPF, ASP.net}
    1. Add new project (Create) to solution.
    2. If project is not SCS then In Solution Add link to SCS source project.
      Solution Explorer +/ Right Click +/ Add project link +/ In dialog choose SCS project as source
  2. For each platform in {WP, M4A, SL, WF, WPF, ASP.net}
    1. Add EXE project (entry-point, Main) to PL layer
    2. Optionally: Add Unit testing assemblies - DLL project to BL layer
