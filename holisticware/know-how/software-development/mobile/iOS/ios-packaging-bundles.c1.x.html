iOS packaging - Bundles

Every iOS (iPhone and iPad) app (and similar for apps on OS X) is a "bundle" which isn't an executable in the traditional sense, but actually a folder hierarchy inside of which app binary resides (along with resources, settings, etc.).

iOS apps are sandboxed (uebersandboxed), so apps are restricted to the complete file-system (shared folders, etc) like for ordinary desktop development (common Documents folder that lives under a user's home folder to which applications have access).
Instead, every app has its own folder hierarchy that's like its own personal set of the folders that would typically be shared across apps.

One of the possible ways to see the file-system structure is with simulator (but not entirely the same)
The folders iOS simulator uses for app installs, settings and app file-system structure is usually: 

	~/Library/Application Support/iPhone Simulator

This is usually divided into subfolders according to installed SDKs: 

	~/Library/Application Support/iPhone Simulator

IPA

IPA size 2012-10:

  1. ipa size > 20 MiB - users are asked to download it only on the Wi-Fi network and the 3G download would be disabled.
  2. max size for app store is 2 GiB.

Make app less than 2gb, then download the remaining data on demand (once the app is launched).
Enterprise builds, have no limits.
