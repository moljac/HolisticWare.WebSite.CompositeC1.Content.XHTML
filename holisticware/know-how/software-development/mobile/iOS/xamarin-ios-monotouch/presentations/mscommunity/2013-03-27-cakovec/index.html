MVC

MVC - Details 1

  • Model
    • Business Logic - POCO (plain old c# objects)
    • data management, encapsulates data/properties and behavior/methods
  • View
    • Objects for building Presentation Layer/User Interface
    • members of UIKit
    • every Control/Widget/View inherits from UIView
    • Capability of drawing on the canvas/presenting
    • respond to user interaction
    • Initialization != drawing presenting
      To show object add as subview to existing UI
    • UIWindow - top level UI object,
      UIViewControllers
    • All Views are children (contained in) Window thus building hierarchy.

MVC - Details 2

  • Controller
    • core of the application
    • handles interaction between UI layer (presentation) and POCO
    • contains most of the code and functionality
    • Tasks - responsibilities
      1. initialization and positioning of the UIView objects in UI
      2. layout and rotation change management
      3. event management - touch, swipe, shakes ...
      4. propagating events and user interactions to Models (Business Logic)
      5. responding to events from Model and refreshing UI (presentation)
    • controller instances have UIView property for top-level UIView with UI content
  • usage: first one to display (it's contained UIView) is created in AppDelegate instance

Xamarin.iOS - MonoTouch

Internals

  1. compiles c# => MSIL => native ARM code
    AOT, ahead of time compiler ~ ngen.exe
  2. **native** applications for iPhone, iPad, iPod
    MSIL is compiled like objective-c into native code (ARM)
  3. dynamic features not allowed (by Apple) - no Code.Emit and similar
  4. runtime is embedded into package/bundle
    influence on the package size
  5. linker - optimizing unused code (removal)
    both on SDK and user assemblies
    IL code is compiled to native and removed from managed assemblies
    only metadata is pertained in assemblies

Xamarin.iOS

Internals - Restrictions

  1. Apple restrictions (kernel) - not available (forbidden)
    • no jitter - everything is compiled
      Ahead of Time Compilation (static compilation)
      similar like manually invoking ngen.exe
    • no dynamic code generation (Dynamic Language Runtime - DLR + DynamicObject)
      CodeDom - Code.Emit
  2. Limited - reduced
    • Reflection
    • Memory Management
    • PInvoke

Xamarin.iOS

Internals - Garbage Collection

  1. iOS uses RetainCounts
    RetainCounts : ReferenceCounts : MemoryPointers != Dealloc Pointers
  2. MonoTouch runtime handles Garbage collection
    handled by managed code embedded in bundle/package
    AutoRelease is created for everything on main thread and derived child threads
