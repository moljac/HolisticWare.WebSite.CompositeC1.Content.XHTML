Local Storage

Internal storage (/data)

/data is your internal storage device: 

	/dev/block/mmcblk0p10 /data ext4 rw,nosuid,nodev,noatime,barrier=1,data=ordered,noauto_da_alloc 0 0

The only reliable directory is /data/data/Your.Package.Name, e.g. System.Environment.GetFolderPath(System.Environment.SpecialFolder.MyDocuments) (which is /data/data/Your.Package.Name/files).

The problem here is that the /data partition is frequently small, so it is not recommended (and possible) to store (download) large amounts of data there.

SD card

SD card is in fact mounted at /mnt/sdcard: 

/dev/block/vold/259:3 /mnt/sdcard vfat rw,dirsync,nosuid,nodev,noexec,noatime,nodiratime,uid=1000,gid=1015,fmask=0002,dmask=0002,allow_utime

tmpfs

`tmpfs` is the filesystem type, and in this case means that the contents will be removed when the filesystem is unmounted. Thus, it is not desirable to store anything on it anyway.

Exceptions

In 2012-05 it seems that some device vendors (Samsung) are mounting SD cards on other mountpoints rather than /mnt/sdcard.
