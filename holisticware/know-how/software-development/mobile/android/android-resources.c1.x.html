Android Resources

Naming

MonoForAndroid
Resources (Layout definitions [a]xml) with multiple extensions (more than 1 dot in the name) are not allowed!
filename.ext2.ext1.ext
Use '_' instead
filename_ext2_ext1.ext

Layout Subfolders

2012-01 for complex projects subfolders in layout folder are not supported.

Possible workarounds:

  • Linking files on filesystem:
    • Windows (Vista+ - Vista and above) 
      MKLINK [[/D] | [/H] | [/J]] Link Target

		/D      Creates a directory symbolic link.  Default is a file
				symbolic link.
		/H      Creates a hard link instead of a symbolic link.
		/J      Creates a Directory Junction.
		Link    specifies the new symbolic link name.
		Target  specifies the path (relative or absolute) that the new link
				refers to.
    • Unixoides 
      ln -s
  • Linking files in IDE:
    • Visual Studio [Mono for Android]
      1. Organize files in subfolders on filesystem
      2. In Layout Directory Use:
        Project +/ Add Existing Item +/

Code Behind in Layout folder

Adding c# source file (.cs) in Resources\Layout folder causes error: 

	Error	7	Resource entry Activityname is already defined.	activityname.axml.cs	1	ProjectName

Cause:
Adding anything (with Add New Item or copy/paste) into Resources\Layout directory changes items Build Action to AndroidResource. This is defined in Visual Studio plugin.

Workaround:
After adding in Properties of the item (.cs) file change Build Action to Compile, or use HolisticWare Project Items for Mono 4 Android.
