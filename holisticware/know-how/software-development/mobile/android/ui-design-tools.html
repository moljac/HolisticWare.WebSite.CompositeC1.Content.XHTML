UI design tools

There are no integrated Visual Designers (WYSIWYG), so integration in Visual Studio is done through External Tools.
For flexible integration External Tools need command line options/arguments of called application.
For more info about External Tools used by HolisticWare team see:
/HolisticWare/Know-How/development/visual-studio/external-tools.aspx

DroidDraw

Good tool, standalone and web, but no commandline (2011-06), so team has filed feature request to get commandline option for loading [a]xml layout.
DroidDraw issues archive: Commandline options (http://code.google.com/p/droiddraw/issues/detail?can=2&q=223&id=223)
In this case integration in Visual Studio no biggy.
Commandline accepts only *.xml, so the java code in 2 files was done in order to accept *.axml files...
