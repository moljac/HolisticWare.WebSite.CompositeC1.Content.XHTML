Prepared everything for deployment with Android SDK and AVD Manager. Checked/downloaded: Available Packages +/ Google USB Drivers.

Tried to install 2 devices on Windows Vista:

  • HTC Sensation
  • Sony Ericsson Xperia X10

in Computer +/ Manage +/ Device Manger drivers were reported

  • for HTC Sensation as "Android Device"
  • Sony Ericsson Xperia X10 as "SEMC HUSB"

Downloaded drivers for:

  • HTC Sensation no drivers, but HTC Sync Application http://www.htc.com/www/help/
  • Sony Ericsson Xperia X10 drivers http://developer.sonyericsson.com/wportal/devworld/search-downloads/driver?cc=gb&lc=en

Driver installation for X10 failed, but for XTC Sensation after installation of Sync application everything worked as charm. Next step was to give it a try and download similar application for Sony Ericsson and got PC Companion http://www.sonyericsson.com/cws/support/mobilephones/downloads/subject/pccompanion/1.689743?cc=gb&lc=en .
Again after installation everything worked like a charm...

Conclusion

It seems that it makes sense to install appropriate syncing applications for devices, because most likely that they have up-to-date USB drivers for device.

USB debugging in Mono for Android

USB debugging is available since version 1.0.1.

In AndroidManifest.xml file, add android:debuggable="true" to the <application> element.

On device set:

  • Device +/ Settings +/ Applications +/ Unknown sources = true
  • Device +/ Settings +/ Applications +/ Development +/ USB debugging = true

Verify that device is connected by executing 

	adb devices

from SDK platform-tools/ directory. If connected, the device name is listed as a "device."

For IDE (Visual Studio/Eclipse), run or debug application as usual. Device Chooser Dialog will be presented that lists the available emulator(s) and connected device(s). Select the target device where application will be deployed.

If using the Android Debug Bridge (adb), it is possible to issue commands with the -d flag to target certain connected device.
