Version Detection

Version is detected with mandroid CLI app.

> C:\Program Files\Android\android-sdk-windows\platform-tools>cd "%PROGRAMFILES%\MSBuild\Novell\" > C:\Program Files\MSBuild\Novell>mandroid.exe --version Oops; > but in the folder there is file called Version with content 4.1.1 and date April 25 >

Windows

Calling command: 

>cd "%PROGRAMFILES%\MSBuild\Novell\"
>mandroid.exe --version

will work on OS X...but that won't help on Windows!!! Because, mandroid.exe is a winexe, and thus can't write to the terminal.

Redirect to file works:

Otherwise in the same folder there is file called Version (file timestamp should be latest installation) with version in it.
The Version file also contains the version information, and is used by IDEs to enable version-specific functionality. User shouldn't delete it, but user can rely on the contents to contain the version number.
[re]moving this file and doing mandroid.exe --verision does not regenerate the file.
