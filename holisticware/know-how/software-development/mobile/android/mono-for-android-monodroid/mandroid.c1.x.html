The `mandroid.exe` program _was_ the "driver" of .apk creation, responsible for creating Android Callable Wrappers, compiling Java source into .class files, compiling .class files into a .dex file, generating the .apk, etc.

Most of this has since been moved into MSBuild tasks in order to speed up the build process.

The `mandroid.exe` program is a /target:winexe app (in contrast to console which is /target:exe), so that it doesn't pop up Console windows whenever build is performed, as that's annoying.

Generally, it should be ignored; one should never invoke it manually, unless wants to grab the version information from `mandroid --version`, and the new Version file will be an easier way to obtain that information.
