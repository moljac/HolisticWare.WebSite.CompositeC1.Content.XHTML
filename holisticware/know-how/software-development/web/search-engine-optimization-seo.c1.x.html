Steps

  • title
    • accurate
    • unique
    • brief, but descriptive
  • description
    • accurate
    • unique
    • reasonable length
  • url structure
    • friendly/static (use words)
    • not to deep
    • single version to reach document
    • lowercase
  • navigation
    • sitemaps
      • sitemap.html for users
      • sitemap.xml for crawlers
      • naturally flowing hierarchy
      • use text for navigation
      • have useful 404 page
  • quality content and services
    • easy to read text
      • spell check!
      • no text images for text
    • organized around topic/title!
    • structure text semantically using tags
    • fresh unique content is desired
    • users before search engines
  • structuring
    • anchor texts
      • quality - descriptive
      • format to distinguish from normal text (CSS)
      • consider internal links too
      • security
        • do not expose mails through mailto (spam harvesting protection) your own and others!
    • heading tags
      • emphasize content
      • use sparingly (maybe single h1 similar to title)
  • image optimization
    • distinctive attributes
      • source - filename
      • alt when images used as links
    • discouraged using images instead of links
    • optimize filenames for image search brief, but descriptive
    • directory structure for images
      • separate (makes CDN possible for speed optimizations)
      • single directory
    • supported file types (png, jpeg, gif, bmp)
    • supply sitemap for images
  • control indexing
    • robots.txt (root directory of site)
    • .httaccess
    • meta tag robots value="NOINDEX"
    • beware of caveats
  • links
    • consider nofollow for spamming
  • mobile
    • prepare for mobile (CSS)
    • notify google!
      • verify that it is indexed by google
        • search with site: operator
    • redirect
      • appropriately
  • submission of sitemaps to search engines
    • engines
      • google
      • yahoo
      • bing

      • google http://www.google.com/webmasters/tools

It is interesting today to look a little more about the criteria that take into account the engine to reference a site in its index.

Links

Bing-like links within the content rather than lists of links-style directory
It is preferable that the site which is an external link is on the same theme as the site link
-More weight given to sites of authority

Domain Names

Bing-NDD, who loves the keyword or phrase referred in its name (remember: the shorter the better)
-Taking into account the age of the site: Bing gives more weight to the old fields

Contents

-Preference to pages with lots of content. So out sites with few lines
-Over-weight to pages with more than 300 words
-Use relevant keywords in the content, without abusing
-Cover one topic per page
-Do no plagiarism, content must be unique
-Bing-penalizes sites that have hidden text

Bounce Rate corrections

From http://searchengineland.com/two-simple-rules-for-fixing-high-bounce-rate-pages-35125 It is usually measured in two ways:

  • The percentage of website visitors who see just one page on your site.
    (definition used by: Google Analytics, WebTrends, Yahoo! Web Analytics, CoreMetrics...)
    a single hit (usually page view) in a visitor's session will mark it as a bounce.
  • The percentage of website visitors who stay on the site for a small amount of time (usually few seconds or less).
    Aggressive definition

Possible causes (exceptions):

  • a one page website
  • offline value proposition is so compelling that people would see just one single webpage and get all the information they need and leave.
