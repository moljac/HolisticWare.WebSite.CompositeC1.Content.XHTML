	You need to give full access to the user running your webapplication to the ~/App_Data folder.

	Since all data and configurations when using xml-store is located in files on the webserver your 
	backup should be complete by doing backup of your applications root folder. If you want to be a 
	bit more picky i believe the folders ~/App_Data, ~/Frontend and ~/Composite/InstalledPackages 
	should do the job.

	For migrating content (using xml files) you copy the ~/App_Data/Composite/DataStores folder 
	from one machine to another. Be aware that it will overwrite any existing data and cannot 
	be used for merging changes.

	Personally i have checked the following into my svn when doing development (list is not exhaustive)

		* ~/App_Data/Composite/Composite.config
		* ~/App_Data/Composite/Configuration/DynamicXmlDataProvider.config
		* ~/App_Data/Composite/DynamicTypeForms
		* ~/App_Data/PageTemplates
		* ~/App_Data/Xslt
		* ~/Frontend
