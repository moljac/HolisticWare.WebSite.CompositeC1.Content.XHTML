HolisticWare Process for websites based on Composite C1

Definitions

  • Template site:
    Composite C1 site that has:
    • passed initialization:
      • admin data
      • localization/internationalization (console language)
      • website type (bare bones, xhtml based, razor based, master page based)
    • Installed some of the packages.
  • Deployment site
    Website for particular customer (additional packages installed, content)

Steps

Some information can be found ont Composite C1 forums: http://compositec1.codeplex.com/discussions/360288

  1. Basic steps (both for template site and deployment site)
    1. Determine whether Website [project] or WebApplication project
      • If WebApplication project then convert WebSite to WebApplication
    2. Basic initial SEO stuff:
      1. canonicalization site with and without www should point to the same
        2 cases
        • http://www.holisticware.net
        • http://holisticware.net
        The best solution is with 301 Moved permanently either redirects or url rewrites. (Note: be aware that without 301 Options:
          Options
        • Redirects - bad!
          Might cause duplicate content!
        IIS howto
      2. URLs lowercase
        http://docs.composite.net/Getting-started/Configuration/UrlFormatting
        In ~/App_Data/Composite/Composite.config uncomment UrlFormatters "LowerCase" 
        <configuration> 
  <Composite.Core.Urls defaultPageUrlProviderName="Default"> 
	<PageUrlProviders> 
	  <add name="Default" type="Composite.Plugins.Routing.Pages.DefaultPageUrlProvider, Composite" /> 
	</PageUrlProviders> 
	<UrlFormatters> 
	  <add name="StringReplace" type="Composite.Plugins.Routing.UrlFormatters.StringReplaceUrlFormatter" mandatory="true"> 
		<ReplacementRules> 
		  <add oldValue=" " newValue="-" /> 
		  <add oldValue="?" newValue="" /> 
		  <add oldValue="#" newValue="" /> 
		</ReplacementRules> 
	  </add> 
	  <add 
		name="LowerCase" 
		type="Composite.Plugins.Routing.UrlFormatters.ToLowerCaseUrlFormatter" 
		mandatory="true" 
		/> 
	  <add 
		name="Aplhabets" 
		type="Composite.Plugins.Routing.UrlFormatters.StringReplaceUrlFormatter" 
		mandatory="true" 
		rulesFile="~/App_Data/Composite/Configuration/UrlFormatting.xml" 
		/> 
	</UrlFormatters> 
  </Composite.Core.Urls> 
</configuration>
      3. URLs with special characters
        add StringReplace (is called by default) add parameters only in ~/App_Data/Composite/Configuration/UrlFormatting.xml

    4. package install (batch install)
    5. reset initialization setting for Template page
      remove: ~\App_Data\Composite\Configuration\InstallationInformation.xml
      Ensure that 'template site' does not have the file ~\App_Data\Composite\Configuration\InstallationInformation.xml (the InstallationInformation.xml will be auto created on new sites the first time they run and this ensure sites individual IDs).
      The alternative is that you create a C1 Package that roll in the content, templates etc. The C1 Package approach has the clear advantage that you can roll it across versions.
