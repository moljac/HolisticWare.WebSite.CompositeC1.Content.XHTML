Enable Composite C1 for additional languages

Composite C1 console (Content Editors, Admins, Developers) open up: 

System > / > App_Data > Composite > DataStores > Composite.Data.Types.IWhiteListedLocale_Published.xml

Visual Studio (for Developers) go to file: 

~\App_Data\Composite\DataStores\Composite.Data.Types.IWhiteListedLocale_Published.xml

where ~ is application/website root.

Quick solution

In the file: 

~\App_Data\Composite\DataStores\Composite.Data.Types.IWhiteListedLocales_public.xml

add lines for the cultures what you need, for example: 

<WhiteListedLocale CultureName="hr-HR" />
<WhiteListedLocale CultureName="ja-JP" />
<WhiteListedLocale CultureName="zh-CN" />

Restart your console.

Able to add languages above on the
System => Languages => Add language

Switching between languages

Case 1:
Let say we start with english website, but decide to switch to german at some point in time?

Press "edit" on the language, and edit url mappings, setting "en" for English, and "" to German will do it

Setting deafualt language in System perspective
Language > Set as default.

It has 2 effects.

  1. Default language is console default language, thus each new page will be added in defalt language (when user is logged in console).
  2. When accessing website there will be a redirect to the first root page that belongs to the default language, unless there other settings (f.e. host name bindings) when you access the website only by a host name.

Url configuration for localization/globalization

Url configuration enables customizing urls like: 

		http://ip/					// for German
		http://ip/en/				// for English
