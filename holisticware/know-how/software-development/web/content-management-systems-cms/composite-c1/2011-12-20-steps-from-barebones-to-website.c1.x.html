If necessary (for example, a lot of rule entries), the rules can be moved to an external file. The file must be placed in /App_Data/Composite/Configuration/ and specified in Composite.config
  1. Installation
    1. Grab latest Composite C1 version from:
    2. Solution
      1. create new (non existing)
      2. add to existing
    3. Add Web Site/Project (Composite C1 WebSite) to Visual Studio Solution
      • rename Website to Website_v.X.Y (where X.Y is Composite C1 version)
      • Check Website =+ Properties =+ Virtual Folder = /
        There might be problems if Virtual folder = /Website_v.X.Y for users without Administrative rights.
    4. Add projects according to HolisticWare project structure:
    • Source Control (SVN)
      1. SVN ignore pattern (un-version and add to ignore pattern-list) 
        	App_Data\Composite\Cache\
	App_Data\Composite\LogFiles\
	App_Data\Composite\ApplicationState\
      2. See details
    • Localization - frontend (Languages)

      http://www.holisticware.net/HolisticWare/Know-How/development/Web/CMS/Composite-C1/2011-03-15-Composite-C1-Localization-Globalization.aspx

      1. Add/Enable language 
        	%PATH%\Website\App_Data\Composite\DataStores\Composite.Data.Types.IWhiteListedLocale_Published.xml

      2. Insert 

        	<WhiteListedLocaleElements CultureName="hr-HR" />
      3. Add Languages
        System =+ Languages =+ Context (Right Click) =+ Add Language
      4. Set default language
        System =+ Languages =+ Language =+ Context (Right Click) =+ Set as Default
