Website to WebApplication conversion

There is not a lot info about WebSite to Webapplication conversion process and issues one might run into during conversion.
A lot of info is noise and not really confirmed like for example Website project file existence, conversion process issues (failure issues), etc.
This write-up/how-to is result of experience and some stuff might be incorrect, but could help to give some directions when going through this process.

Description

  • Website:
    • Assembly per page (aspx file) and/or Control (ascx) file
    • Compiled into assemblies on-the-fly, when page is accessed (http request)
    • does not rely on project file (it has one, but it is not crucial)
    • Enables live editing and deploying subtrees (pages in folders on the file system) of the site.
      Changes compiled and applied on access (http request)
  • WebApplication:
    • Single assembly - whole app in single dll.
      Does not require loading of bunch of small dlls and keeping track of changes between pages (aspx files), controls (ascx files) and corresponding assemblies (dlls).
    • Whole project must be deployed
    • Recommended for bigger project and for project where pages do not change frequently or content (web structure - tree) is generated dynamically.

Conversion process:

  • Creates/generates:
    1. code behind files (*.cs and designer.cs)
    2. project file (*.csproj) that holds all information about files and build process
  • Fails unexpectedly sometimes and there is not a lot info in Visual Studio about reasons (only pages that failed to convert)
    Some possibilities:
    • Missing directives for Page 
      	<%@ Page attribute="value" [attribute="value"...] %>
      or for Control 
      	<%@ Control attribute="value" [attribute="value" ... ] %>
      Sometimes it suffices to add following directives for Page 
      	<%@ Page  %>
      or for Control 
      	<%@ Control %>
    • Incorrect Register directives
    • Multiple pages or controls sharing same code file or base class, so Visual Studio could not create appropriate code behind files (either *.cs or *.designer.cs).
    • Voodoo-sh1t
      sometimes it just helps: make file dirty (change something and delete) and reconvert!

Steps

  • Add Web Application Project to Solution
    Web Application, MVC Application or Empty Web Application
  • Just to assure there is no leftovers that could mess up conversion and build/compilation process
    Delete temp ASP.net files (just to be sure):
    %USERPROFILE%\AppData\Local\Temp\Temporary ASP.NET Files\*
  • Natural:
    1. In Visual Studio Explorer!!! Copy Website to WebApplication project
      Web Application is not yet Web Application!! Files are not converted!
    2. Issue convert command on WebApplication project to convert Web Application Project to Web Application!!
      Oxymoron!
      Context Menu (Right Click) +/ Convert to Web Application
    3. Rebuild
    4. While build != OK
      1. Rebuild
      2. Fix conversion/build issues in WebApplication project?!?!?!
  • When depending on other projects based on Website (Composite C1):
    1. Fix build and possible conversion issues in Website
      steps above should be done once to see possible problems
    2. In Visual Studio Explorer!!! Copy Website to WebApplication project
      Web Application is not yet Web Application!! Files are not converted!
    3. Issue convert command on WebApplication project to convert Web Application Project to Web Application!!
      Oxymoron!
      Context Menu (Right Click) +/ Convert to Web Application
    4. Rebuild
    5. While build != OK
      1. Rebuild
      2. Fix conversion/build issues in WebApplication project?!?!?!
