Multiproject (Solution) Project templates

Visual Studio Template

For Application Suites

  1. for each Project in Solution
    • export project (File +/ Export Template ... )
    • In Wizard
      • Edit appropriate data
      • Uncheck option: Automatically import in Visual Studio
  2. create root project directory for multi project Solution (Multitargeting solution)
  3. add .vstemplate text/xml file to root folder
    • add to folder some other stuff like
      • icon file for project - Icon.ico
  4. for each exported Project Template
    • extract templates into separate folders
    • move/copy individual folders to root folder where .vstemplate resides
  5. edit .vstemplate xml file
    • use Visual Studio for syntax validation against schema (debugging is quite difficult so be sure that everything in root template is OK)
    • change Type="Project" to "ProjectType"
    • add links relative to each individual file
  6. edit projects that have dependencies usually ASP.net project that depends on Silverlight. If necessary create dummy XAP files (can be empty text file to silence Visual Studio)
