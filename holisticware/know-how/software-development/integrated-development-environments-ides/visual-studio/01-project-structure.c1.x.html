Project and Solution Structure on HolisticWare projects

Recommended Solution structure and Project naming for HolisticWare projects

Proposal for Solution and Project structure on HolisticWare project 2011-09.
Draft
Subject to change! 

	<Company>.<Project>.<AssemblyType>_<OrderId>_<ArchitectureLayer>_<ApplicationType>_<PlatformTechnologyType>
  • AssemblyType := DLL | EXE
    sort order
  • OrderId (nnn) (n is numeric)
    main purpose - sort order for Project Linker in Visual Studio Solution Explorer
    sample: WP7 has currently (2011-09) smallest subset of features, core code located here
    • 0 - Documentation
    • 1 - Data Access
    • 2 - Business Logic
    • 5 - Presentation Layer
    • 8 - Entry point - Main - EXE
  • ArchitectureLayer := DA | BL | PL
    • DA - Data Access
    • BL - Business Logic
    • PL - Presentation Layer
  • ApplicationType := mClnt | dClnt | rClnt | Srvr
    • mClnt - mobile Client
      • iOS - MonoTouch
      • Android - Mono for Android
      • Windows Phone
    • dClnt - desktop Client (fat Client) Windows Forms | WPF
    • rClnt - rich Client (silverligh)
    • Srvr - server application (ASP.net)
  • PlatformTechnologyType - various
    • mClnt
      • M4A
        Mono for Android
      • MTc
        MonoTouch (iOS)
      • WPv
        Windows Phone v - version (WP7, WP71 - Mango)
    • dClnt
      • WF
        Windows Forms
      • WPF
        Windows Presentation Foundation
    • rClnt
      • SLv
        Silverlight v-version
    • Server
      • ASPnetv
        v-version ASPnet35mvc

Steps during creating Projects in Solution

  1. Create projects according to naming recommendations
  2. Change default Namespaces
    Project +/ Properties +/ Default Namespace:
    • Business Logic
      Company.Project.BL
    • Presentation Layer
      Company.Project.PL
    • Entry Point projects (Main) Presentation Layer Launchers
      Company.Project.PL
  3. Source code linking: (Visual Studio Project Linker)
    1. BL
      1. Project based on platform with least features should be 'starting' project which will be extended in various subsequent project. This project is source project in Project Linker terminology to other projects.
        Note: this project is usually mobile (mClnt)
        2011-10 Windows Phone because of restrictions (no network stack, sockets). At this point in time Android and iOS have less imposed restrictions.
    2. PL
