External Tools

Intro

External tools are used for integration with other tools not directly bound with Visual Studio (external appliactions, shells etc).
External tools can be exported and/or imported with other Visual Studio settings.

Integration of UI WYSIWYG Designer tools for Android:

Persisting

Persisting external tools can be made through:

  • registry entries (keys), or
  • Visual Studio settings file *.vssettings.

This is usually useful to avoid tedious typing in GUI after installations and/or for settings sharing by teams...

Visual Studio Settings files (vssettings)

Visual Studio Settings (Tools / Import and Export Settings)

Registry

  • Registry keys
    • Visual Studio 2005 
      HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\VisualStudio\8.0\External Tools\
    • Visual Studio 2008 
      HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\VisualStudio\9.0\External Tools\
    • Visual Studio 2010 
      HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\VisualStudio\10.0\External Tools\

Add keys for example: 

“ToolArg”=”$(ItemPath)”
“ToolOpt”=dword:00000008
“ToolCmd”=”notepad.exe”
“ToolDir”=”$(TargetDir)”

Batch script to backup Registry Entries regarding External Tools: 

set FILENAME=.\exported_machine.reg
set KEY="HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\VisualStudio\10.0\External Tools"

regedit.exe ^
	/e %FILENAME% ^
	%KEY%

REG EXPORT ^
	%KEY% ^
	%FILENAME%
	
set FILENAME=.\exported_user.reg
set KEY="HKEY_CURRENT_USER\SOFTWARE\Microsoft\VisualStudio\10.0\External Tools"

regedit.exe ^
	/e %FILENAME% ^
	%KEY%

REG EXPORT ^
	%KEY% ^
	%FILENAME%
