Projects and file inclusion patterns

Date created: 2013-04-05
Date modified: 2013-04-05

Valuable information for web development especially during conversions of WebSites to Web Application projects like in the case of Composite C1. Namely Websites do not rely on project files.

Other usage is for front-end developers which do not need Visual Studio and its project files, so sometimes files in use by front-end developers might not be part of build (which is controlled by project (csproj) file).

"Include in Project" for folders/directories does not use this (wildcard) approach, but enumerates each file in folder and subfolders and adds it to project file.

For known project (folder/directosy) structures user can modify our project file to automatically include the files below the folders. This can be done with wildcards in Include attribute (references).
Patterns must be added to project file (project file manually edited) once - when folder is created.

Patterns

Note: xml element can be Compile (for cs files), Content (for content), None, etc

  • Single file 
    			<Compile Include="code.cs" />
  • List of files 
    			<Compile Include="code01.cs;code02.cs" />
  • All files in folder 
    			<Content Include="Path\To\Folder\*.*" />
  • All files in folder and subfolders (below a specific folder) 
    			<Content Include="Path\To\Folder\**" />
  • All files with suffix in folder and subfolders 
    			<Content Include="Path\To\Folder\\**\*.suffix" />
  • All files folders with suffix 
    			<Content Include="Path\To\Folder\**\*suffix\*.*" /> 
			
    			<Content Include="Path\To\Folder\**\*suffix\*" />
